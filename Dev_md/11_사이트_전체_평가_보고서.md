## 3. 항목별 상세 평가

### 3.1 아키텍처 & 구조 (87/100) - A

**강점**
- Context 기반 상태 관리 깔끔 분리 (Theme, Language, Auth, Toast)
- Lazy loading + Suspense로 100% 페이지 코드 스플리팅
- Vite `manualChunks`로 vendor/supabase/markdown 번들 분리
- 콘텐츠(content/)와 UI(pages/components/) 명확 분리
- AuthGuard/AdminGuard 라우트 보호 패턴
- MENU_GROUPS 기반 동적 메뉴 구성 시스템

**개선 필요**
- Error Boundary 미구현 - React 에러 시 전체 앱 크래시
- 라우트 순서 이슈: `/community/board/:id`가 `/community/board/write`보다 먼저 매칭될 수 있음
- `ScrollAnimationInit` 빈 컴포넌트 - 비표준 패턴

```
src/
├── components/     # 레이아웃/공유 컴포넌트 (6개)
├── config/         # 설정 파일 (admin, lessons)
├── content/        # 학습 콘텐츠 (62개 .js 파일)
├── contexts/       # React Context (4개: Auth, Theme, Language, Toast)
├── hooks/          # 커스텀 훅 (1개: useScrollAnimation)
├── layouts/        # 레이아웃 (1개: PublicLayout)
├── pages/          # 페이지 컴포넌트 (15개)
├── styles/         # CSS 파일 (16개, ~4,500줄)
└── utils/          # 유틸리티 (4개: auth, supabase, translations, siteConfig)
```

- **ESLint/Prettier 미설정**: 코드 스타일 강제 수단 없음
- **테스트 0%**: Jest/Vitest/RTL 없음

### 3.7 접근성 - a11y (35/100) - D+

**심각한 문제**
- `<button>`에 `aria-label` 거의 없음
  - 햄버거 메뉴, 테마 토글, 색상 선택 등
- 스크린리더용 아이콘 대체 텍스트 없음
- 키보드 트랩 미구현 (모달, 드롭다운)
- Skip navigation 링크 없음
- 폼 `<label>` 요소 없이 placeholder만 사용
- `aria-expanded`, `aria-controls`, `role="menu"` 미적용
- focus 상태 커스텀 정의 부족

**권장 조치 (우선순위)**
1. 모든 인터랙티브 요소에 `aria-label` 추가
2. `<label htmlFor>` 연결 (Login, Register, ForgotPassword)
3. 키보드 네비게이션 (Escape 키 드롭다운 닫기 등)
4. Skip to main content 링크
5. `aria-expanded` 토글 상태 관리
